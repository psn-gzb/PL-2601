Git Reset: Soft Reset & Hard Reset
git reset moves the HEAD pointer backward
But how much data it keeps or deletes is dependents on
--soft
--hard

Git Soft Reset (--soft)
What it does is;
	moves HEAD to previous commit 
	keeps changes in staging area
	keeps the files unchanged (data inside the file will remain)

git reset --soft <commit-id>

Git Hard Reset (--hard)
What it does is;
	moves HEAD to previous commit
	clears the staging area
	deletes working directory changes (data will get deleted)

git reset --hard <commit-id>

Git Revert
It will undo a commit safely by creating a new commit

What git revert does is;
it doesn't delete any commit
	it creates; a new commit, keeps full history

git revert creates a new commit that removes the changes made by another commit

git revert <commit-id>
git revert HEAD
